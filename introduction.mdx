## Why Walrus

Walrus is powerful. Makes your data decentralized and so → verifiable by anyone. Moving to Walrus from any centralized service provider means: 

1. You don't depend on any service provider anymore. To read and write data you can use locally installed Walrus tooling. For this you'll need a Sui wallet with tokens (SUI & WAL). You'll also need some RAM and CPU: Walrus de-/encodes your file in memory and that is a resource demanding process. It also runs hundreds of HTTP requests to decentralize your file over the network.
2. Your data is verifiable. This also means it can now be read by anyone (!). This is important: do not throw your private data on decentralized storage just like that. Humans may read it, robots may index it, AI agents can download it. If you want to keep something decentralised and private → encrypt it before storing and keep the key safe.
3. You decide for how long the file is on the network (Walrus epochs) and pay accordingly in WAL tokens. You also manage the file lifecycle: extend or delete the file. 
4. From your perspective (a Walrus client), your files are now a key-value store. Any content on Walrus gets a unique identifier (either blobID or quiltPatchID for batched uploads). You read the content using that identifier. As simple as that

## Why Tusky

Tusky is an infra provider for [Sui Walrus](https://walrus.xyz). It makes using Walrus easy at any scale.

Tusky gives you managed infra — shared or dedicated publishers and aggregators, encryption, access control, and a familiar file system like experience on top of the Walrus network.

## How It Works

Tusky orchestrates (dedicated or shared) **publishers** and **aggregators**  for each user:

- **Publishers** handle encoding and writing your data to the Walrus network. Each user gets a dedicated managed wallet for publishing, funded from your prepaid balance.
- **Aggregators** reconstruct and serve your data from the decentralized network. Tusky runs optimized aggregators with standard URL schemes for seamless access.

You interact with Tusky through the [dashboard](https://app.tusky.io), the [TypeScript SDK](/sdk/getting-started), or the [REST API](/api-reference/overview). Under the hood, Tusky handles blob encoding, storage node coordination, encryption key management, and data retrieval.

<Note>
  Tusky is not a storage network — it is an **infrastructure layer** built on top of Walrus. Your data is stored on the decentralized Walrus network. Tusky provides the tooling, encryption, and access management to make that storage practical for real applications.
</Note>
