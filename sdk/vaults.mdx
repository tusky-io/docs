Vaults are the primary organizational unit in Tusky. Each vault acts as a secure container for files, with its own encryption settings, access policies, and member list. The `tusky.vault` module provides full CRUD operations and member management.

## Create a Vault

Create a new vault with a name and optional configuration:

```typescript
const vault = await tusky.vault.create("Project Assets", {
  encrypted: true,
  description: "Design files and brand assets",
  defaultEpochs: 5,
  autoExtend: true,
});

console.log("Created vault:", vault.id);
```

### Options

| Parameter | Type | Default | Description |
| --- | --- | --- | --- |
| `encrypted` | `boolean` | `true` | Enable client-side encryption for all files in this vault. |
| `description` | `string` | — | Human-readable description of the vault. |
| `defaultEpochs` | `number` | `3` | Default number of Walrus storage epochs for uploads. |
| `autoExtend` | `boolean` | `false` | Automatically renew storage before epoch expiry. |
| `tags` | `string[]` | — | Tags for filtering and organization. |

<Warning>
  The `encrypted` setting is permanent and cannot be changed after the vault is created.
</Warning>

## Get a Vault

Retrieve a vault by its ID:

```typescript
const vault = await tusky.vault.get("vault_abc123");

console.log(vault.name);         // "Project Assets"
console.log(vault.encrypted);    // true
console.log(vault.fileCount);    // 42
console.log(vault.totalSize);    // 1073741824 (bytes)
```

The returned vault object includes metadata such as file count, total size, creation date, and member count.

## List Vaults

Retrieve all vaults in your account:

```typescript
const vaults = await tusky.vault.list();

for (const vault of vaults) {
  console.log(`${vault.name} — ${vault.fileCount} files`);
}
```

<Tip>
  The list is ordered by creation date (newest first) by default. You can use the returned metadata to build vault selectors, dashboards, or migration tools.
</Tip>

## Update a Vault

Modify a vault's mutable settings:

```typescript
const updated = await tusky.vault.update("vault_abc123", {
  name: "Brand Assets 2025",
  description: "Updated brand guidelines and logos",
  defaultEpochs: 10,
  autoExtend: true,
});

console.log("Updated:", updated.name);
```

<Note>
  You can update the name, description, default epochs, auto-extend setting, and tags. The encryption type and vault ID are immutable.
</Note>

## Delete a Vault

Permanently remove a vault and its Tusky metadata:

```typescript
await tusky.vault.delete("vault_abc123");
console.log("Vault deleted");
```

<Warning>
  Deleting a vault removes all Tusky metadata and access control. Files already published to Walrus remain on the network until their storage epochs expire, but they will no longer be served through your Tusky gateway.
</Warning>

## Complete Example

```typescript
import { Tusky } from "@tusky-io/ts-sdk";

const tusky = new Tusky({ apiKey: process.env.TUSKY_API_KEY });

// Create
const vault = await tusky.vault.create("Demo Vault", {
  encrypted: false,
  description: "Public demo content",
});

// Read
const fetched = await tusky.vault.get(vault.id);
console.log("Vault:", fetched.name, "| Encrypted:", fetched.encrypted);

// Update
await tusky.vault.update(vault.id, {
  description: "Updated description",
});

// List all
const allVaults = await tusky.vault.list();
console.log("Total vaults:", allVaults.length);

// Delete
await tusky.vault.delete(vault.id);
console.log("Cleaned up");
```
