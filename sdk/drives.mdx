Drives are the primary organizational unit in Tusky. Each drive acts as a secure container for files, with its own encryption settings, access policies, and member list. The `tusky.drive` module provides full CRUD operations and member management.

## Create a Drive

Create a new drive with a name and optional configuration:

```typescript
const drive = await tusky.drive.create("Project Assets", {
  encrypted: true,
  description: "Design files and brand assets",
  defaultEpochs: 5,
  autoExtend: true,
});

console.log("Created drive:", drive.id);
```

### Options

| Parameter | Type | Default | Description |
| --- | --- | --- | --- |
| `encrypted` | `boolean` | `true` | Enable client-side encryption for all files in this drive. |
| `description` | `string` | — | Human-readable description of the drive. |
| `defaultEpochs` | `number` | `3` | Default number of Walrus storage epochs for uploads. |
| `autoExtend` | `boolean` | `false` | Automatically renew storage before epoch expiry. |
| `tags` | `string[]` | — | Tags for filtering and organization. |

<Warning>
  The `encrypted` setting is permanent and cannot be changed after the drive is created.
</Warning>

## Get a Drive

Retrieve a drive by its ID:

```typescript
const drive = await tusky.drive.get("drive_abc123");

console.log(drive.name);         // "Project Assets"
console.log(drive.encrypted);    // true
console.log(drive.fileCount);    // 42
console.log(drive.totalSize);    // 1073741824 (bytes)
```

The returned drive object includes metadata such as file count, total size, creation date, and member count.

## List Drives

Retrieve all drives in your account:

```typescript
const drives = await tusky.drive.list();

for (const drive of drives) {
  console.log(`${drive.name} — ${drive.fileCount} files`);
}
```

<Tip>
  The list is ordered by creation date (newest first) by default. You can use the returned metadata to build drive selectors, dashboards, or migration tools.
</Tip>

## Update a Drive

Modify a drive's mutable settings:

```typescript
const updated = await tusky.drive.update("drive_abc123", {
  name: "Brand Assets 2025",
  description: "Updated brand guidelines and logos",
  defaultEpochs: 10,
  autoExtend: true,
});

console.log("Updated:", updated.name);
```

<Note>
  You can update the name, description, default epochs, auto-extend setting, and tags. The encryption type and drive ID are immutable.
</Note>

## Delete a Drive

Permanently remove a drive and its Tusky metadata:

```typescript
await tusky.drive.delete("drive_abc123");
console.log("Drive deleted");
```

<Warning>
  Deleting a drive removes all Tusky metadata and access control. Files already published to Walrus remain on the network until their storage epochs expire, but they will no longer be served through your Tusky aggregator.
</Warning>

## Complete Example

```typescript
import { Tusky } from "@tusky-io/ts-sdk";

const tusky = new Tusky({ apiKey: process.env.TUSKY_API_KEY });

// Create
const drive = await tusky.drive.create("Demo Drive", {
  encrypted: false,
  description: "Public demo content",
});

// Read
const fetched = await tusky.drive.get(drive.id);
console.log("Drive:", fetched.name, "| Encrypted:", fetched.encrypted);

// Update
await tusky.drive.update(drive.id, {
  description: "Updated description",
});

// List all
const allDrives = await tusky.drive.list();
console.log("Total drives:", allDrives.length);

// Delete
await tusky.drive.delete(drive.id);
console.log("Cleaned up");
```
