Gateways are the retrieval layer of Tusky — they fetch your content from the Walrus network and serve it over standard HTTPS. Every Tusky user gets a **private gateway** that serves only their own uploaded content, complete with CDN edge caching, DDoS protection, and optional custom domains.

## What is a gateway?

In the Walrus ecosystem, an **aggregator** is a service that retrieves blob data from Walrus storage nodes and assembles it for the requesting client. Tusky wraps this aggregator functionality into a fully managed **gateway** service that adds access control, caching, analytics, and custom domain support on top.

<CardGroup cols={2}>

## Your gateway URL

When you create a Tusky account, a private gateway is automatically provisioned with a unique subdomain:

```
https://<your-subdomain>.gateway.tusky.io
```

This URL is immediately usable — no configuration required. You can find your gateway URL in the Tusky dashboard under **Settings → Gateway**.

<Tip>
  You can connect a **custom domain** (e.g., `cdn.yourdomain.com`) to your gateway for a branded experience. See the [Custom Domains](/gateways/custom-domains) guide for setup instructions.
</Tip>

## Standard URL schemes

Tusky gateways follow a standard URL scheme for retrieving content. All files are accessible using either their **blob ID** or their **quilt patch ID**.

<Tabs>
  <Tab title="By blob ID">
    Retrieve a file (or an entire quilt) using its Walrus blob ID:

    ```
    GET /v1/blobs/{blobId}
    ```

    **Example:**

    ```
    https://acme.gateway.tusky.io/v1/blobs/5Gk9pM3Nh...
    ```

    Use this when you have the raw Walrus blob identifier — for standalone files or when accessing a full quilt.
  </Tab>
  <Tab title="By quilt patch ID">
    Retrieve an individual file from a quilt using its quilt patch ID:

    ```
    GET /v1/blobs/by-quilt-patch-id/{quiltPatchId}
    ```

    **Example:**

    ```
    https://acme.gateway.tusky.io/v1/blobs/by-quilt-patch-id/qp_8Xr2nZ...
    ```

    Use this to address a specific file within a batched quilt without downloading the entire quilt.
  </Tab>
</Tabs>

<Info>
  Both URL schemes return the raw file data with the correct `Content-Type` header, making them suitable for direct embedding in web pages, API responses, and applications.
</Info>

## Gateway architecture

```
┌────────────┐     ┌────────────────┐     ┌──────────────┐     ┌─────────────┐
│   Client    │────▶│  CDN / Edge    │────▶│  Tusky       │────▶│   Walrus    │
│  (browser)  │◀────│  Cache         │◀────│  Gateway     │◀────│   Network   │
└────────────┘     └────────────────┘     └──────────────┘     └─────────────┘
```

1. **Client request** — A user or application requests content via your gateway URL.
2. **CDN check** — If the content is cached at an edge location, it's returned immediately with minimal latency.
3. **Gateway fetch** — On a cache miss, the Tusky gateway verifies the blob belongs to your account, then fetches it from Walrus.
4. **Walrus retrieval** — The gateway contacts Walrus storage nodes, reassembles the erasure-coded data, and returns the file.
5. **Cache and respond** — The response is cached at the edge and delivered to the client.

## Private gateway access control

Unlike public Walrus aggregators, your Tusky gateway enforces **account-scoped access control**:

- Only blobs published through your managed wallet are served.
- Requests for blob IDs not associated with your account return a `404 Not Found`.
- This prevents your gateway (and custom domain) from being used to serve arbitrary Walrus content.

<Note>
  If you need to serve content publicly without account scoping (e.g., for a fully open data project), you can use a public Walrus aggregator directly with your blob IDs. Your Tusky gateway is designed for controlled, account-scoped delivery.
</Note>

## Custom domains and subdomains

You can map your own domain or subdomain to your Tusky gateway:

| Feature | Details |
|---|---|
| **Custom subdomain** | Point `cdn.yourdomain.com` to your gateway via CNAME. |
| **Apex domain** | Use an ALIAS or ANAME record for root domains (provider-dependent). |
| **Automatic TLS** | Tusky provisions and renews TLS certificates automatically. |
| **Multiple domains** | Attach more than one domain to a single gateway. |

For step-by-step instructions, see [Custom Domains](/gateways/custom-domains).

## Analytics

Every gateway includes built-in analytics accessible from the Tusky dashboard:

- **Request volume** — Total requests over time.
- **Bandwidth** — Data transferred per period.
- **Top content** — Most-requested blob IDs and files.
- **Geographic distribution** — Where your users are accessing content from.
- **Cache hit rate** — Percentage of requests served from CDN cache.

For details, see [Gateway Analytics](/gateways/analytics).
