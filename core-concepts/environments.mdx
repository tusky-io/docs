Environments are the top-level containers in Tusky for organizing and managing your files. Think of them as secure folders or cloud storage buckets — each environment groups related files together and provides granular control over encryption, access, and sharing.

## Environment types

Tusky supports two environment types, each suited to different use cases.

<CardGroup cols={2}>
  <Card title="Public environment" icon="globe">
    Files stored in a public environment are uploaded **unencrypted** to Walrus. Anyone with the blob ID or aggregator URL can retrieve them. Ideal for open data, public websites, and shared media.
  </Card>
  <Card title="Encrypted environment" icon="lock">
    Files are **encrypted client-side** before leaving your device. Only environment members with the correct decryption key — derived from their connected Sui wallet or managed through Seal — can read the contents.
  </Card>
</CardGroup>

<Warning>
  A environment's encryption setting is permanent. You cannot convert a public environment to an encrypted environment (or vice-versa) after creation.
</Warning>

## Creating an environment

<Steps>
  <Step title="Choose a name and type">
    Give your environment a descriptive name and select either **Public** or **Encrypted**. This choice is immutable.
  </Step>
  <Step title="Configure settings">
    Set default storage duration (in epochs), enable auto-extend if desired, and optionally add a description or tags.
  </Step>
  <Step title="Start uploading">
    Once the environment is created you can immediately upload files, add members, or connect the environment to an aggregator.
  </Step>
</Steps>

## Environment settings

Every environment exposes a settings panel where you can manage:

| Setting | Description |
|---|---|
| **Name & description** | Human-readable metadata for the environment. |
| **Default storage epochs** | The number of Walrus storage epochs applied to new uploads unless overridden per-file. |
| **Auto-extend** | When enabled, Tusky automatically renews storage before expiry so your data is never lost. |
| **Encryption** | Displays the encryption status (read-only after creation). |

## Members and roles

Environments support collaborative access through a role-based membership model.

<Tabs>
  <Tab title="Owner">
    Full control over the environment — can upload, download, delete files, manage members, and modify settings. Each environment has exactly one owner (the creator).
  </Tab>
  <Tab title="Manager">
    Can upload, download, and delete files and invite or remove Viewers. Cannot change environment settings or remove other Managers.
  </Tab>
  <Tab title="Viewer">
    Read-only access — can download and view files but cannot upload, delete, or manage membership.
  </Tab>
</Tabs>

<Info>
  For encrypted environments, adding a member shares the environment's decryption capability with that member's Sui wallet through Seal. Members never see raw encryption keys.
</Info>

## Environment lifecycle

Understanding the full lifecycle of an environment helps you plan storage and collaboration effectively.

```
Create → Configure → Upload files → Share / add members → Manage → Archive / Delete
```

1. **Create** — A new environment is provisioned and linked to your account.
2. **Active** — Files are uploaded, accessed, and managed. Members collaborate as needed.
3. **Archive** *(optional)* — You can stop new uploads while keeping existing files accessible for their remaining storage duration.
4. **Delete** — Removing an environment deletes all Tusky metadata. Files already published to Walrus remain available until their storage epochs expire; however, they will no longer be served through your Tusky aggregator.

<Note>
  Deleting an environment does **not** immediately delete data from the Walrus network. Walrus blobs persist until their purchased storage epochs expire. If you have ejected blobs to your own wallet, those blobs remain fully under your control regardless of environment status.
</Note>
