---
title: List Drives
api: 'GET https://api.tusky.io/v2/drives'
---

List all drives accessible to the authenticated user. Results are returned in reverse chronological order and paginated using cursor-based pagination.

## Query Parameters

<ParamField query="cursor" type="string">
  Opaque cursor from a previous response to fetch the next page of results.
</ParamField>

<ParamField query="limit" type="number" default="20">
  Number of drives to return per page. Minimum `1`, maximum `100`.
</ParamField>

## Response

<ResponseField name="data" type="array">
  Array of drive objects.

  <Expandable title="Drive object properties">
    <ResponseField name="id" type="string">
      Unique identifier for the drive.
    </ResponseField>

    <ResponseField name="name" type="string">
      The name of the drive.
    </ResponseField>

    <ResponseField name="encrypted" type="boolean">
      Whether the drive is encrypted.
    </ResponseField>

    <ResponseField name="createdAt" type="string">
      ISO 8601 timestamp of when the drive was created.
    </ResponseField>

    <ResponseField name="updatedAt" type="string">
      ISO 8601 timestamp of when the drive was last updated.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="nextCursor" type="string">
  Cursor to pass as a query parameter to retrieve the next page. `null` if there are no more results.
</ResponseField>

<ResponseField name="hasMore" type="boolean">
  Whether there are more results beyond this page.
</ResponseField>

## Example

<CodeGroup>

```bash cURL
curl "https://api.tusky.io/v2/drives?limit=10" \
  -H "Authorization: Bearer YOUR_API_KEY"
```

</CodeGroup>
