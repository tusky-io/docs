---
title: Get Aggregator
api: 'GET https://api.tusky.io/v2/aggregators/{aggregatorId}'
---

Retrieve an aggregator by ID.

## Path Parameters

<ParamField path="aggregatorId" type="string" required>
  The aggregator ID. Use `default` for the account's primary aggregator.
</ParamField>

## Response

<ResponseField name="id" type="string">
  Unique aggregator identifier.
</ResponseField>

<ResponseField name="url" type="string">
  The full URL of the aggregator (e.g., `https://myapp.mytusky.xyz`).
</ResponseField>

<ResponseField name="subdomain" type="string">
  The subdomain (e.g., `myapp`).
</ResponseField>

<ResponseField name="customDomain" type="string">
  Custom domain, or `null` if not configured.
</ResponseField>

<ResponseField name="restricted" type="boolean">
  Whether authentication is required.
</ResponseField>

<ResponseField name="restrictedMode" type="string">
  How clients pass the access token when `restricted` is `true`. One of: `header`, `query`, or `path`.
</ResponseField>

<ResponseField name="public" type="boolean">
  When `true`, serves any content from the Walrus network. When `false`, serves only your account's content.
</ResponseField>

<ResponseField name="cache" type="boolean">
  Whether Cloudflare edge caching is enabled. When `true`, content is cached at 300+ edge locations for low-latency delivery.
</ResponseField>

<ResponseField name="fallbackToBackup" type="boolean">
  Whether the aggregator falls back to the centralized backup store when the Walrus network is unavailable.
</ResponseField>

<ResponseField name="analyticsEnabled" type="boolean">
  Whether analytics tracking is enabled.
</ResponseField>

<ResponseField name="createdAt" type="string">
  ISO 8601 creation timestamp.
</ResponseField>

## Example

<CodeGroup>

```bash cURL
curl https://api.tusky.io/v2/aggregators/default \
  -H "Api-Key: YOUR_API_KEY"
```

</CodeGroup>
