---
title: Create Volume
api: 'POST https://api.tusky.io/v2/volumes'
---

Create a new volume. Volumes are containers for decentralized files and folders. They can be optionally encrypted.

## Request Body

<ParamField body="name" type="string" required>
  Volume name. 1–128 characters.
</ParamField>

<ParamField body="encrypted" type="boolean" default="false">
  Whether to enable end-to-end encryption. When `true`, all files are encrypted client-side before upload. Encryption keys are managed via Sui Seal and stored as secrets. **Cannot be changed after creation.**
</ParamField>

<ParamField body="publisherIds" type="array">
  Array of publisher IDs to assign to this volume. Assigning multiple publishers enables **horizontal scaling** — uploads are distributed across publishers for shorter publishing times. If omitted, the account's default publisher is used.
</ParamField>

## Response

<ResponseField name="id" type="string">Volume ID.</ResponseField>
<ResponseField name="name" type="string">Volume name.</ResponseField>
<ResponseField name="encrypted" type="boolean">Whether encrypted.</ResponseField>
<ResponseField name="publisherIds" type="array">Assigned publisher IDs.</ResponseField>
<ResponseField name="createdAt" type="string">ISO 8601 creation timestamp.</ResponseField>

## Example

<CodeGroup>

```bash cURL
curl -X POST https://api.tusky.io/v2/volumes \
  -H "Api-Key: YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Production Assets",
    "encrypted": false,
    "publisherIds": ["pub_primary", "pub_secondary"]
  }'
```

</CodeGroup>

### Response

```json
{
  "id": "vol_abc123",
  "name": "Production Assets",
  "encrypted": false,
  "publisherIds": ["pub_primary", "pub_secondary"],
  "createdAt": "2026-03-07T14:00:00Z"
}
```

<Note>
  The `encrypted` setting is **permanent** — it cannot be changed after creation.
</Note>

<Tip>
  Assigning multiple publishers to a volume scales your publishing infrastructure horizontally. Incoming uploads are load-balanced across all assigned publishers, reducing publishing latency for high-throughput workloads.
</Tip>
