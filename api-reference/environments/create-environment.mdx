---
title: Create Environment
api: 'POST https://api.tusky.io/v2/environments'
---

Create a new environment to organize and store files on the Walrus network.

## Request Body

<ParamField body="name" type="string" required>
  The name of the environment. Must be between 1 and 128 characters.
</ParamField>

<ParamField body="encrypted" type="boolean" default="false">
  Whether to enable end-to-end encryption for the environment. When enabled, all files uploaded to this environment are encrypted before being stored on Walrus. This cannot be changed after creation.
</ParamField>

## Response

<ResponseField name="id" type="string">
  Unique identifier for the environment.
</ResponseField>

<ResponseField name="name" type="string">
  The name of the environment.
</ResponseField>

<ResponseField name="encrypted" type="boolean">
  Whether the environment is encrypted.
</ResponseField>

<ResponseField name="createdAt" type="string">
  ISO 8601 timestamp of when the environment was created.
</ResponseField>

<ResponseField name="updatedAt" type="string">
  ISO 8601 timestamp of when the environment was last updated.
</ResponseField>

## Example

<CodeGroup>

```bash cURL
curl -X POST https://api.tusky.io/v2/environments \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "My First Environment",
    "encrypted": false
  }'
```

</CodeGroup>

### Response

```json
{
  "id": "env_abc123def456",
  "name": "My First Environment",
  "encrypted": false,
  "createdAt": "2025-07-01T12:00:00.000Z",
  "updatedAt": "2025-07-01T12:00:00.000Z"
}
```

<Note>
  The `encrypted` setting is permanent and cannot be changed after environment creation. Choose carefully based on your security requirements.
</Note>
