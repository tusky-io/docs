---
title: Create Environment
api: 'POST https://api.tusky.io/v2/environments'
---

Create a new environment. Environments are containers for files, folders, and secrets. They can be public or private (encrypted).

## Request Body

<ParamField body="name" type="string" required>
  Environment name. 1–128 characters.
</ParamField>

<ParamField body="encrypted" type="boolean" default="false">
  Whether to enable end-to-end encryption. When `true`, all files are encrypted client-side before upload. Encryption keys are managed via Sui Seal and stored as secrets. **Cannot be changed after creation.**
</ParamField>

<ParamField body="publisherId" type="string">
  Publisher to use for this environment. If omitted, the account's default publisher is used. See [Publishers](/api-reference/publishers/create-publisher).
</ParamField>

## Response

<ResponseField name="id" type="string">Environment ID.</ResponseField>
<ResponseField name="name" type="string">Environment name.</ResponseField>
<ResponseField name="encrypted" type="boolean">Whether encrypted.</ResponseField>
<ResponseField name="publisherId" type="string">Assigned publisher ID.</ResponseField>
<ResponseField name="createdAt" type="string">ISO 8601 creation timestamp.</ResponseField>

## Example

<CodeGroup>

```bash cURL
curl -X POST https://api.tusky.io/v2/environments \
  -H "Api-Key: YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Production Assets",
    "encrypted": false,
    "publisherId": "pub_archival"
  }'
```

</CodeGroup>

### Response

```json
{
  "id": "env_abc123",
  "name": "Production Assets",
  "encrypted": false,
  "publisherId": "pub_archival",
  "createdAt": "2026-03-07T14:00:00Z"
}
```

<Note>
  The `encrypted` setting is **permanent** — it cannot be changed after creation. Choose based on your security requirements.
</Note>
