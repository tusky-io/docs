---
title: Get Secret
api: 'GET https://api.tusky.io/v2/secrets/{path}'
---

Read a secret at the specified path. The encrypted data is fetched from the Seal network and decrypted client-side.

## Path Parameters

<ParamField path="path" type="string" required>
  The path of the secret (e.g., `production/database`).
</ParamField>

## Query Parameters

<ParamField query="version" type="number">
  Specific version to retrieve. Defaults to the latest.
</ParamField>

## Response

<ResponseField name="path" type="string">The secret path.</ResponseField>
<ResponseField name="data" type="object">Key-value pairs containing the secret data.</ResponseField>
<ResponseField name="version" type="number">Version number.</ResponseField>
<ResponseField name="managed" type="boolean">`true` for Tusky-managed secrets (e.g., volume encryption keys), `false` for user-created.</ResponseField>
<ResponseField name="createdAt" type="string">ISO 8601 timestamp.</ResponseField>

## Example

<CodeGroup>

```bash cURL — user secret
curl https://api.tusky.io/v2/secrets/production/database \
  -H "Api-Key: YOUR_API_KEY"
```

```bash cURL — managed encryption key
curl https://api.tusky.io/v2/secrets/tusky/volumes/vol_abc123/encryption \
  -H "Api-Key: YOUR_API_KEY"
```

</CodeGroup>

### Response

```json
{
  "path": "production/database",
  "data": {
    "host": "db.example.com",
    "port": "5432",
    "username": "app_user",
    "password": "s3cur3_p4ssw0rd"
  },
  "version": 1,
  "managed": false,
  "createdAt": "2026-03-06T14:00:00Z"
}
```

<Note>
  Managed secrets at `tusky/volumes/{envId}/encryption` contain encryption keys for private volumes. These are created automatically when an encrypted volume is created and **cannot be deleted**.
</Note>
