---
title: Put Secret
api: 'PUT https://api.tusky.io/v2/secrets/{path}'
---

Create or update a secret at the specified path. Secrets are global to your account. The data is encrypted using [Sui Seal](https://seal-docs.wal.app) — Tusky never has access to the plaintext values.

## Path Parameters

<ParamField path="path" type="string" required>
  The path to store the secret at. Use `/` separators for hierarchy (e.g., `production/database/password`).
</ParamField>

## Request Body

<ParamField body="data" type="object" required>
  Key-value pairs to store. Values are strings.
</ParamField>

## Response

<ResponseField name="path" type="string">The full path of the stored secret.</ResponseField>
<ResponseField name="version" type="number">Version number. Increments on each update.</ResponseField>
<ResponseField name="managed" type="boolean">`false` for user-created secrets.</ResponseField>
<ResponseField name="createdAt" type="string">ISO 8601 timestamp.</ResponseField>

## Example

<CodeGroup>

```bash cURL
curl -X PUT https://api.tusky.io/v2/secrets/production/database \
  -H "Api-Key: YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "data": {
      "host": "db.example.com",
      "port": "5432",
      "username": "app_user",
      "password": "s3cur3_p4ssw0rd"
    }
  }'
```

</CodeGroup>

### Response

```json
{
  "path": "production/database",
  "version": 1,
  "managed": false,
  "createdAt": "2026-03-06T14:00:00Z"
}
```

<Note>
  Secrets are encrypted client-side using [Sui Seal](https://seal-docs.wal.app). Access is controlled by on-chain policies.
</Note>

<Warning>
  You cannot write to paths under `tusky/` — these are reserved for **managed secrets** created automatically by Tusky (e.g., encryption keys for private environments).
</Warning>
