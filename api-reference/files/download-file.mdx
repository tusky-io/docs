---
title: Download File
api: 'GET https://api.tusky.io/v2/files/{fileId}/download'
---

Download the binary content of a file. The response returns the raw file data with the appropriate `Content-Type` header.

## Path Parameters

<ParamField path="fileId" type="string" required>
  The unique identifier of the file to download.
</ParamField>

## Response

The response body contains the raw file binary. The response includes these headers:

| Header              | Description                                         |
| ------------------- | --------------------------------------------------- |
| `Content-Type`      | The MIME type of the file (e.g., `application/pdf`) |
| `Content-Length`    | The size of the file in bytes                       |
| `Content-Disposition` | Attachment with the original filename             |

## Example

<CodeGroup>

```bash cURL
curl https://api.tusky.io/v2/files/file_mno345pqr678/download \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -o document.pdf
```

</CodeGroup>

### Response

```
HTTP/1.1 200 OK
Content-Type: application/pdf
Content-Length: 1048576
Content-Disposition: attachment; filename="document.pdf"

<binary file data>
```

<Note>
  For files stored in encrypted vaults, decryption is handled server-side automatically. The response returns the decrypted file content.
</Note>
