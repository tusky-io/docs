---
title: Estimate Upload Cost
api: 'POST https://api.tusky.io/v2/uploads/estimate'
---

Estimate the storage cost for one or more files before uploading. Accepts a list of file sizes and returns a per-file breakdown plus totals in both WAL and SUI.

Use this to preview costs, build budgeting tools, or validate that your balance is sufficient before starting an upload batch.

## Request Body

<ParamField body="files" type="array" required>
  List of file size objects to estimate. Each entry represents one file.

  <Expandable title="File size object">
    <ParamField body="size" type="number" required>
      File size in bytes.
    </ParamField>
    <ParamField body="name" type="string">
      Optional label for this file — returned in the response for easy matching.
    </ParamField>
  </Expandable>
</ParamField>

<ParamField body="epochs" type="number" default="Publisher default">
  Number of Walrus storage epochs to estimate for. Overrides the publisher default. Range: 1–100.
</ParamField>

<ParamField body="publisherId" type="string">
  Optional publisher ID. When provided, the estimate uses the publisher's `epochs` and `quilt` settings. When omitted, the account default publisher is used.
</ParamField>

## Response

<ResponseField name="files" type="array">
  Per-file cost breakdown.
  <Expandable title="File estimate object">
    <ResponseField name="name" type="string">
      The label from the request, or `"file_0"`, `"file_1"`, etc. if not provided.
    </ResponseField>
    <ResponseField name="size" type="number">
      File size in bytes.
    </ResponseField>
    <ResponseField name="encodedSize" type="number">
      Walrus-encoded size in bytes (includes erasure coding overhead).
    </ResponseField>
    <ResponseField name="walCost" type="string">
      WAL cost for this file.
    </ResponseField>
    <ResponseField name="gasCost" type="string">
      Estimated SUI gas cost for this file.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="totals" type="object">
  Aggregate totals across all files.
  <Expandable title="Totals object">
    <ResponseField name="fileCount" type="number">
      Number of files in the estimate.
    </ResponseField>
    <ResponseField name="totalSize" type="number">
      Combined file size in bytes.
    </ResponseField>
    <ResponseField name="totalEncodedSize" type="number">
      Combined Walrus-encoded size in bytes.
    </ResponseField>
    <ResponseField name="totalWal" type="string">
      Total WAL cost.
    </ResponseField>
    <ResponseField name="totalGas" type="string">
      Total estimated SUI gas cost.
    </ResponseField>
    <ResponseField name="totalSui" type="string">
      Total SUI needed (gas only — WAL is a separate token).
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="epochs" type="number">
  The epoch count used for this estimate.
</ResponseField>

<ResponseField name="balance" type="object">
  Current wallet balances and whether they cover the estimate.
  <Expandable title="Balance object">
    <ResponseField name="walBalance" type="string">
      Current WAL balance.
    </ResponseField>
    <ResponseField name="suiBalance" type="string">
      Current SUI balance.
    </ResponseField>
    <ResponseField name="sufficient" type="boolean">
      Whether current balances cover the estimated cost.
    </ResponseField>
    <ResponseField name="walDeficit" type="string">
      WAL shortfall (0 if sufficient).
    </ResponseField>
    <ResponseField name="suiDeficit" type="string">
      SUI shortfall (0 if sufficient).
    </ResponseField>
  </Expandable>
</ResponseField>

## Example

<CodeGroup>

```bash cURL
curl -X POST https://api.tusky.io/v2/uploads/estimate \
  -H "Api-Key: YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "files": [
      { "size": 10485760, "name": "video.mp4" },
      { "size": 2097152, "name": "document.pdf" },
      { "size": 524288, "name": "image.png" }
    ],
    "epochs": 5
  }'
```

</CodeGroup>

<Tip>
  Use the `balance.sufficient` field to check whether the upload batch can proceed without a top-up. If `autoConvert` is enabled on the publisher, a WAL deficit can be covered automatically by swapping SUI → WAL.
</Tip>

<Note>
  Estimates are based on current Walrus network rates and may fluctuate. Actual costs are determined at transaction time. The `encodedSize` includes Walrus erasure coding overhead (~5x the original size).
</Note>
