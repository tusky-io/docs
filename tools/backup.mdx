Tusky offers a **centralized backup** service that stores copies of your files in a traditional blobstore alongside Walrus. This provides an additional layer of insurance against data loss — your files exist on both the decentralized Walrus network and a reliable centralized backup.

## Why Backup?

The Walrus network is designed for durability through redundant encoding across storage nodes. However, some use cases demand an extra safety net:

<CardGroup cols={2}>
  <Card title="Human Error" icon="clock-rotate-left">
    Mistakes happen: deleting files by a missclick, disabling auto-extend flag, missing blob expiration notifications...
  </Card>

  <Card title="Peace of Mind" icon="heart">
    For mission-critical data, a backup in a proven centralized system provides additional confidence.
  </Card>
</CardGroup>

## How It Works

<Steps>
  <Step title="Enable backup on a volume">
    Turn on backup for any volume through the dashboard or API. Backup is configured per volume — you can choose which volumes need the extra protection.
  </Step>
  <Step title=" Once enabled">
    Tusky runs over all your files from given volume and downloads it to backup.
  </Step>
  <Step title="Automatic on upload">
    Every file uploaded to the volume is automatically copied to Tusky's centralized blobstore in addition to being published to Walrus.
  </Step>
  <Step title="Once removed from the network">
    When your file gets deleted from Walrus, Tusky will still hold it in backup for 2 epochs
  </Step>
  <Step title="Transparent access">
    File retrieval works exactly the same whether backup is enabled or not. If the primary Walrus path is unavailable, the aggregator can fall back to the backup copy - only if enabled on your aggregator level.
  </Step>
</Steps>

## Enabling Backup

<Tabs>
  <Tab title="Dashboard">
    Navigate to your volume's **Settings → Backup** and toggle **Enable Backup**. All future uploads to this volume will be backed up automatically.
  </Tab>
  <Tab title="SDK">
    Enable backup when creating or updating a volume:

    ```typescript
    // Enable on creation
    const volume = await tusky.volume.create("Critical Data", {
      encrypted: true,
      backup: true,
    });
    
    // Enable on existing volume
    await tusky.volume.update("vol_abc123", {
      backup: true,
    });
    ```
  </Tab>
  <Tab title="API">
    ```bash
    curl -X PATCH https://api.tusky.io/v1/volumes/vol_abc123 \
      -H "Authorization: Bearer tsk_live_abc123" \
      -H "Content-Type: application/json" \
      -d '{"backup": true}'
    ```
  </Tab>
</Tabs>

## Disabling Backup

You can disable backup at any time:

- New uploads will no longer be backed up.
- Existing backup copies are retained for the remainder of the current billing period and then deleted.

<Warning>
  Disabling backup does not immediately delete existing backup copies. They are retained until the end of the billing period. After that, only the Walrus copies remain.
</Warning>
